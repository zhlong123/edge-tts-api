API 调用01

本服务是 OpenAI 兼容的 Edge TTS 端点。任何支持自定义 base_url 的客户端,把它指向 http://127.0.0.1:9880 即可直接对接。

⚠ 调用的是微软 Edge 内置的"朗读"接口,未获官方授权。商业化推荐 Azure Speech 或本地 TTS。

鉴权02

默认关闭。开启后请求需带:

http
Authorization: Bearer sk-edge-tts-local

用 python manage.py toggle-auth on 开关,api_key 在 config/settings.json 改。

合成端点03

POST /v1/audio/speech

OpenAI 兼容的核心合成端点,返回 audio/mpeg。

bash
curl -X POST http://127.0.0.1:9880/v1/audio/speech \
  -H "Authorization: Bearer sk-edge-tts-local" \
  -H "Content-Type: application/json" \
  -d '{"input":"你好","voice":"zh-CN-XiaoxiaoNeural"}' \
  --output hello.mp3

字段:input(必填)、voice(默认 zh-CN-XiaoxiaoNeural)、speed(0.25 - 4.0,默认 1.0)、response_format(仅支持 mp3)。model 任意值,当前不区分 tts-1 / tts-1-hd。

Python 示例04

python
from openai import OpenAI

client = OpenAI(
    api_key="sk-edge-tts-local",
    base_url="http://127.0.0.1:9880/v1",
)
resp = client.audio.speech.create(
    model="tts-1",
    voice="zh-CN-XiaoxiaoNeural",
    input="你好,世界",
)
with open("hello.mp3", "wb") as f:
    f.write(resp.read())

其它端点05

  • GET /health — 健康检查,返回 { status, default_voice, port }
  • GET /v1/voices — 300+ Edge 公网全部音色(网页下拉用这个)
  • GET /v1/voices/presets — 14 个精选预设(本地,离线可用)
  • GET /v1/models — OpenAI 兼容的模型列表

常见问题06

合成返回 502 "Edge TTS 合成失败"?

Edge TTS 走公网,需能访问微软服务器。开 VPN / 代理可能反而不通(被 SNI 拦截),直连国内网络即可。

局域网设备连不上?

检查本机防火墙:Windows 防火墙入站规则放行 TCP 9880;Linux ufw allow 9880/tcp。

怎么加新音色?

编辑 config/voice_presets.json。voice 字段须为 GET /v1/voices 返回的 ShortName。
